Packet Tracer Network Design

Full network topology built in Cisco Packet Tracer

Project Overview

This project simulates a secure small business network using Cisco Packet Tracer. It models remote site connectivity using an IPsec VPN tunnel to ensure encrypted communication across different subnets. The setup includes routers, switches, and end devices, with static routing configured to enable subnet-to-subnet traffic while preserving network segmentation.

Key Features

  • 🟢 IPsec VPN configured between two or more remote routers
  • 🟡 IP addressing scheme: 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24
  • 🔵 Static routing between subnets over the VPN tunnel
  • 🔐 End devices securely communicate using encrypted channels

Challenges & Lessons

Configuring the IPsec VPN was crucial for secure inter-site traffic. I learned how to verify encrypted connections and troubleshoot routing issues with `ping` and `tracert`. This exercise deepened my understanding of VPN architecture, static routing, and subnet design.
